# M2 — neovim-плагин: навигация

## Цель

Появляется первый собственный neovim-плагин в составе проекта. После M2 при курсоре на markdown-ссылке вида `[text](path)` нажатие настроенного keymap'а открывает целевой файл — в текущем окне или в вертикальном сплите. Путь скрывается через conceal: видна только часть `text`, подчёркнутая, как гиперссылка; при заходе курсора на строку путь разворачивается.

После M2 базовый цикл «увидел ссылку — перешёл по ней» работает в neovim без копирования путей руками. На этот фундамент M3 навешивает picker для **создания** ссылок, M4 — поиск по тегам, M5 — `:MdxRename`.

## Что входит и что не входит

### Входит

- Lua-плагин в подкаталоге `nvim/` репозитория `mdx`, рядом с `lsp/`.
- Парсер ссылки `[text](target)` под курсором на основе treesitter: курсор-адресуемая позиция определяется через `vim.treesitter.get_node`, восхождение по родителям до узла `inline_link` (язык `markdown_inline`).
- Резолвер `target` в абсолютный путь файловой системы: разворачивание `~`, разрешение относительно каталога текущего буфера, нормализация.
- Команды `:MdxFollow` и `:MdxFollowSplit`, открывающие целевой файл соответственно в текущем окне (`:edit`) или в вертикальном сплите (`:vsplit`).
- Conceal-конфигурация через treesitter query, скрывающая `[`, `]`, `(`, `link_destination`, `)` внутри узла `inline_link`. Подсветка текста ссылки наследуется от стандартной группы `@markup.link.label`. Активируется только в `markdown`-буферах.
- Функция `require('mdx').setup(opts)` для пользовательской конфигурации: keymaps, поведение open-команд, включение/выключение conceal.
- Plugin-spec для lazy.nvim с конфигурацией по умолчанию и явной зависимостью от `nvim-treesitter` с парсерами `markdown` и `markdown_inline`.
- Юнит-тесты на парсер и резолвер через `plenary.nvim`.

### Не входит (отложено)

- Picker по существующим заметкам и команда `:MdxInsertLink` — это M3.
- Поиск по тегам и команда `:MdxTagSearch` — это M4.
- `:MdxRename` — это M5.
- Подсветка содержимого ссылки в виде иконки или префикса перед `text`.
- Поддержка `#section`-якорей внутри целевого файла (открытие файла со скроллом до раздела).
- Image-embed (`![alt](path)`) на M2 трактуется так же, как обычная ссылка; preview-функциональность не предусмотрена.
- Регистрация ex-команд `:MdxFollow` и `:MdxFollowSplit` через `vim.api.nvim_create_user_command` — публичный API представлен функциями `require('mdx').follow()` и `require('mdx').follow_split()`; ex-команды добавляются как тонкая обёртка в шаге 4.

## Технические решения

| Вопрос | Решение |
|---|---|
| Расположение плагина | Подкаталог `nvim/` в репозитории `mdx`, рядом с `lsp/`. Совместная эволюция плагина и LSP-сервера упрощает разработку и обновление. При необходимости публикации плагин выделяется в отдельный репозиторий механически — публичный API под `lua/mdx/` это переживёт. |
| Минимальная версия neovim | `0.11+`. Обоснование: проект уже использует `vim.lsp.config` API из этой ветки; treesitter-API в этой версии стабилен. |
| Парсер ссылок | Treesitter-узел `inline_link` (язык `markdown_inline`). Поиск через `vim.treesitter.get_node({ pos = { row, col } })` + восхождение по родителям. Дочерние узлы `link_text` и `link_destination` дают `text` и `target`. Ссылки внутри fenced code blocks парсером не классифицируются как `inline_link` — за счёт этого conceal и follow корректно их пропускают. |
| Conceal | Treesitter query в `after/queries/markdown_inline/highlights.scm` с директивой `; extends`. Скрытие узлов `[`, `]`, `(`, `link_destination`, `)` через `(#set! conceal "")`. Подсветка `link_text` уже задана дефолтными query как `@markup.link.label` и линкуется со стандартной группой ссылки. |
| Скоуп активации | Только `filetype=markdown`. Реализуется через `ftplugin/markdown.lua` — стандартный механизм neovim для filetype-локальных настроек. |
| Тестовый фреймворк | `nvim-lua/plenary.nvim` (`PlenaryBustedDirectory`). Это де-факто стандарт для тестирования lua-плагинов. |
| Default keymaps | Префиксная группа `<leader>m` в markdown-буферах: `<leader>mf` → `MdxFollow`, `<leader>ms` → `MdxFollowSplit`. Группа задумана как контейнер для будущих команд M3–M5 (`<leader>mi` insert link, `<leader>mt` tag search, `<leader>mr` rename). Пользователь может переопределить или отключить любой keymap через `setup({ keymaps = ... })`. |
| Поведение при отсутствии ссылки под курсором | `MdxFollow()` и `MdxFollowSplit()` выводят `vim.notify("mdx: no link under cursor", vim.log.levels.INFO)` и возвращают `false`. Никакой обёртки с fallback на стандартное поведение клавиши не требуется — keymap'ы лежат в выделенной leader-группе и не пересекаются со встроенными. |
| Не-`file` ссылки (URL) | Пропускаются с уведомлением через `vim.notify("mdx: external URL, ignored", vim.log.levels.INFO)`. Открытие в браузере — отдельная фича вне M2. |

## Шаги выполнения

Шаги упорядочены так, чтобы после каждого можно было запустить осмысленную проверку — либо тест, либо ручной сценарий в neovim.

### Шаг 1. Скаффолдинг плагина

Создать каталог `nvim/` с минимальной структурой и заглушкой публичного API.

#### Состав каталога `nvim/`

`lua/mdx/init.lua` объявляет публичные функции:

```lua
function M.setup(opts)
function M.follow()
function M.follow_split()
```

`setup(opts)` принимает таблицу пользовательских настроек. На этом шаге тело функции пустое — мердж с дефолтами добавляется в Шаге 6. `follow()` и `follow_split()` возвращают `false` как заглушка; реальная логика появляется в Шаге 4. Цель шага — проверить, что плагин подгружается.

`ftplugin/markdown.lua` оставляется пустым — содержательно заполняется в Шаге 5.

Каталог `after/queries/markdown_inline/` создаётся пустым; query-файл наполняется в Шаге 5.

#### Установка через lazy.nvim

В `nvim/README.md` приводится spec для локальной разработки:

```lua
{
    dir = "/home/oleg/projects/mdx/nvim",
    name = "mdx",
    ft = "markdown",
    dependencies = {
        {
            "nvim-treesitter/nvim-treesitter",
            opts = {
                ensure_installed = { "markdown", "markdown_inline" },
            },
        },
    },
    config = function()
        require("mdx").setup({})
    end,
}
```

**Готовность:** после `:Lazy reload mdx` команда `:lua print(vim.inspect(require('mdx')))` печатает таблицу с тремя функциями, ошибок в `:messages` нет. `:checkhealth nvim-treesitter` подтверждает наличие парсеров `markdown` и `markdown_inline`.

### Шаг 2. Парсер ссылки под курсором

Реализовать функцию, извлекающую markdown-ссылку, в чьих пределах находится курсор, через treesitter.

#### Состав `lua/mdx/link.lua`

```lua
function M.under_cursor() -- returns table or nil
```

Алгоритм:

1. Получить позицию курсора через `vim.api.nvim_win_get_cursor(0)`. Привести к 0-based: `row = row - 1`.
2. Получить узел: `vim.treesitter.get_node({ pos = { row, col }, ignore_injections = false })`. Флаг `ignore_injections = false` обязателен — без него поиск идёт только по основному `markdown`-дереву, а узлы `inline_link` живут в injected `markdown_inline` и остаются невидимыми.
3. Восходить по родителям через `node:parent()` до узла с типом `inline_link`. Если такой не найден — вернуть `nil`.
4. Среди дочерних узлов найти `link_text` и `link_destination`. Если хотя бы одного нет — вернуть `nil`.
5. Извлечь текст узлов через `vim.treesitter.get_node_text(child, 0)`.
6. Получить диапазон контейнерного `inline_link` через `node:range()`; вернуть таблицу.

Возвращаемая структура:

```lua
{
    text   = "...",   -- содержимое link_text
    target = "...",   -- содержимое link_destination
    start  = N,       -- 0-based колонка начала "["
    finish = N,       -- 0-based колонка после ")"
}
```

#### Тесты

`tests/link_spec.lua` использует `vim.api.nvim_buf_set_lines` для подготовки буфера, вручную выставляет `vim.bo.filetype = "markdown"`, дожидается parsing'а через `vim.treesitter.get_parser():parse()` и позиционирует курсор через `vim.api.nvim_win_set_cursor`. Покрывает кейсы:

- одна ссылка на строке, курсор на ней;
- несколько ссылок на строке, курсор внутри второй;
- курсор не на ссылке;
- ссылка с title (`[text](target "title")`);
- ссылка внутри fenced code block — должна возвращать `nil`.

**Готовность:** `nvim --headless -c "PlenaryBustedDirectory nvim/tests/" -c "qa!"` выполняется без ошибок.

### Шаг 3. Резолвер пути

Реализовать функцию, превращающую `target` ссылки в абсолютный путь файловой системы.

#### Состав `lua/mdx/resolve.lua`

```lua
function M.target_to_path(target, source_dir) -- returns string or nil
```

Поведение:

1. Если `target` начинается с одной из URL-схем (`http://`, `https://`, `mailto:`, `ftp://`, `tel:`, `#`) — возвращает `nil`. Эти случаи отдельно обрабатываются вызывающим кодом.
2. Если `target` начинается с `~/` или равен `~` — разворачивает через `vim.fn.expand("~")`, конкатенирует остаток.
3. Если `target` начинается с `/` — возвращает как есть после нормализации `vim.fs.normalize`.
4. Иначе — резолвит относительно `source_dir`: `vim.fs.normalize(source_dir .. "/" .. target)`.

`source_dir` передаётся вызывающим кодом и обычно равен `vim.fs.dirname(vim.api.nvim_buf_get_name(0))`.

#### Тесты

`tests/resolve_spec.lua` — табличный тест на пары `(target, source_dir, expected)` для случаев: относительный путь с `..`, абсолютный путь, путь с `~`, URL-схема (ожидается `nil`), якорь `#section` (ожидается `nil`).

**Готовность:** все табличные кейсы зелёные.

### Шаг 4. Команды `MdxFollow` и `MdxFollowSplit`

Связать парсер и резолвер в единый сценарий открытия файла.

#### Состав `lua/mdx/init.lua`

```lua
function M.follow()        -- returns boolean
function M.follow_split()  -- returns boolean
```

Поведение:

1. Вызывают `link.under_cursor()`. Если `nil` — выводят `vim.notify("mdx: no link under cursor", vim.log.levels.INFO)` и возвращают `false`.
2. Вызывают `resolve.target_to_path(link.target, source_dir)`, где `source_dir = vim.fs.dirname(vim.api.nvim_buf_get_name(0))`. Если `nil` — выводят `vim.notify("mdx: external URL, ignored", vim.log.levels.INFO)` и возвращают `true` (сценарий обработан).
3. Открывают файл соответствующей командой:
    - `M.follow()` — `vim.cmd.edit(vim.fn.fnameescape(path))`.
    - `M.follow_split()` — `vim.cmd.vsplit(vim.fn.fnameescape(path))`.
4. Возвращают `true`.

Дополнительно: в `lua/mdx/init.lua` объявить ex-команды через `vim.api.nvim_create_user_command("MdxFollow", M.follow, {})` и `vim.api.nvim_create_user_command("MdxFollowSplit", M.follow_split, {})` внутри `setup()`.

**Готовность:** при курсоре на `[Test](./test.md)` в открытом `.md` файле выполнение `:MdxFollow` открывает `test.md` в текущем окне; `:MdxFollowSplit` открывает в вертикальном сплите.

### Шаг 5. Conceal через treesitter

Скрыть оформление ссылки и оставить только `text`, читаемый как гиперссылка.

#### Состав `after/queries/markdown_inline/highlights.scm`

```scheme
; extends

(inline_link
  "[" @conceal (#set! conceal "")
  "]" @conceal (#set! conceal "")
  "(" @conceal (#set! conceal "")
  (link_destination) @conceal (#set! conceal "")
  ")" @conceal (#set! conceal ""))
```

Директива `; extends` означает, что query добавляется к встроенному файлу `markdown_inline/highlights.scm` из `nvim-treesitter`, а не заменяет его. Дефолтные query уже подсвечивают `link_text` как `@markup.link.label`, что в стандартных color schemes отображается подчёркнутым в цвете ссылки — отдельной highlight-настройки не требуется.

#### Состав `ftplugin/markdown.lua`

При загрузке буфера выполняется:

```lua
vim.opt_local.conceallevel  = 2
vim.opt_local.concealcursor = ""
```

`conceallevel=2` включает conceal с замещением; `concealcursor=""` гарантирует, что при заходе курсора на строку весь текст разворачивается полностью.

**Готовность:** при открытии `.md` файла со ссылкой `[Test](./test.md)` отображается только подчёркнутый `Test`; при перемещении курсора на строку видна полная ссылка. Внутри fenced code block литеральная конструкция `[text](path)` отображается без conceal.

### Шаг 6. Конфигурация через `setup()`

Реализовать пользовательскую конфигурацию: keymaps, переключение conceal.

#### Состав `lua/mdx/init.lua`

```lua
local defaults = {
    keymaps = {
        follow       = "<leader>mf",
        follow_split = "<leader>ms",
    },
    conceal = true,
}

function M.setup(opts)
```

`setup(opts)` глубоко мерджит `opts` с `defaults` и сохраняет результат в `M.config`. Никаких побочных эффектов на уровне глобального состояния не выполняет; все настройки применяются per-buffer в `ftplugin/markdown.lua`.

#### Изменения в `ftplugin/markdown.lua`

1. Если `M.config.conceal == false` — пропустить установку `conceallevel`/`concealcursor`. Query-файл из `after/queries/` подгружается всегда, но без `conceallevel=2` он визуально не действует.
2. Для каждого keymap из `M.config.keymaps`, если значение не `false`, навесить buffer-локальный keymap:

```lua
vim.keymap.set("n", M.config.keymaps.follow, function()
    require("mdx").follow()
end, { buffer = true, desc = "mdx: follow link" })

vim.keymap.set("n", M.config.keymaps.follow_split, function()
    require("mdx").follow_split()
end, { buffer = true, desc = "mdx: follow link in vsplit" })
```

`desc` обеспечивает читаемые подписи в `which-key.nvim`, если он установлен пользователем.

**Готовность:** `<leader>mf` на строке со ссылкой открывает целевой файл, на строке без ссылки выводит уведомление. `<leader>ms` открывает в вертикальном сплите. `setup({ conceal = false })` отключает скрытие путей. `setup({ keymaps = { follow_split = false } })` отключает соответствующий keymap.

## Критерии готовности M2

1. `:Lazy reload mdx` подгружает плагин без ошибок в `:messages`. `:checkhealth nvim-treesitter` показывает установленные парсеры `markdown` и `markdown_inline`.
2. `nvim --headless -c "PlenaryBustedDirectory nvim/tests/" -c "qa!"` зелёный.
3. В открытом `.md` файле курсор на `[Test](./test.md)`, `<leader>mf` открывает `test.md` в текущем окне; `<leader>ms` — в вертикальном сплите.
4. На строке без ссылки `<leader>mf` выводит `vim.notify` с сообщением `mdx: no link under cursor` и не делает ничего ещё.
5. Path-часть ссылки скрыта, виден подчёркнутый `text`. При заходе курсора на строку отображается вся ссылка целиком. Внутри fenced code block литеральные `[text](path)` отображаются без conceal.
6. `setup({ conceal = false })` отключает скрытие; `setup({ keymaps = { follow = false } })` отключает соответствующий keymap.
7. README содержит минимальный пример подключения плагина.
