# M3 — neovim-плагин: создание ссылок

## Цель

После M3 в markdown-буфере нажатие `<leader>mi` (или выполнение `:MdxInsertLink`) открывает picker по существующим markdown-заметкам проекта. Выбор заметки вставляет в текущую позицию курсора готовую markdown-ссылку вида `[Заголовок заметки](относительный/путь.md)`. Текст ссылки берётся из первого заголовка `# H1` целевого файла; если такого нет — из имени файла без расширения. Путь — нормализованный относительный, считаемый от каталога текущего буфера.

После M3 пара «создать ссылку → перейти по ссылке» закрыта целиком в neovim. Picker и команда `:MdxFollow` из M2 образуют замкнутый цикл работы со ссылками. M4 поверх этого добавит поиск по тегам, M5 — `:MdxRename`.

## Что входит и что не входит

### Входит

- Telescope-based picker по существующим `.md`-файлам в проекте, отображающий каждую заметку строкой `Заголовок (относительный/путь.md)`.
- Сборщик заметок: рекурсивный обход дерева проекта от выбранного корня с исключением заданных паттернов.
- Извлечение `text`-части ссылки: первый `# H1` из тела заметки; если такого нет — имя файла без расширения.
- Вычисление относительного пути от каталога текущего буфера до выбранной заметки.
- Вставка готовой строки ссылки в позицию курсора.
- Команда `:MdxInsertLink` и keymap `<leader>mi` (по умолчанию).
- Расширение `setup(opts)`: корень поиска, источник `text`, паттерны исключения, keymap.
- Юнит-тесты на сборщик заметок, извлечение заголовка и форматирование ссылки.

### Не входит (отложено)

- Создание новой заметки прямо из picker'а, когда подходящего файла нет в проекте, — отдельная фича, кандидат на M3.5.
- Поиск ссылок по тегам и команда `:MdxTagSearch` — это M4.
- Обновление существующих ссылок при переименовании файла (`:MdxRename`) — это M5.
- Автокомплит ссылок при наборе `[` через `nvim-cmp` или встроенный `omnifunc`.
- Section-anchors в формируемых ссылках (`[text](file.md#section)`).
- Не-`.md` файлы в picker'е (изображения, исходники, конфиги) — picker отбирает только `.md`.
- Альтернативные picker-движки (`fzf-lua`, `snacks.picker`, `vim.ui.select`) — telescope является единственной поддерживаемой бэкенд-зависимостью на M3.

## Технические решения

| Вопрос | Решение |
|---|---|
| Picker engine | Telescope. Уже зависимость основного nvim-dev пользователя, идиоматичен для neovim-плагинов, имеет богатый action API. Интеграция изолирована в `lua/mdx/picker.lua` — на M3 это монолит, при необходимости picker-движок вынесется в подключаемый адаптер позже. |
| Корень поиска заметок | Маркер-based: `vim.fs.root(0, { ".git", ".mdx", "stylua.toml" })`. Если маркеры не найдены — `vim.fn.getcwd()`. Пользователь может переопределить через `setup({ root = "/abs/path" })` или `setup({ root = function() return ... end })` для динамики. |
| Сборщик файлов | `vim.fs.find` с предикатом по расширению `.md`. Возвращает абсолютные пути. Исключения через предикат фильтрации, использующий заранее известные паттерны (`.git/`, `node_modules/`, `_smoke.md`) плюс пользовательский список. |
| Источник `text` ссылки | По умолчанию — первый `# H1` из файла; если такого нет — имя файла без расширения. Чтение первой `# `-строки делается без полной загрузки файла: `io.lines` до первого совпадения. Опция `setup({ link_text = "title"\|"filename"\|"auto" })` фиксирует поведение; `"auto"` — описанное выше. |
| Относительный путь | `vim.fs.relpath(source_dir, target_path)` (доступно с neovim 0.11). Возвращает строку с `../`-откатами и без ведущего `./`. Перед вставкой проверяется, что путь не пустой; иначе используется имя файла. |
| Вставка ссылки в буфер | `vim.api.nvim_put({ link_string }, "c", true, true)`. Параметры: `"c"` — character-wise; четвёртый аргумент `true` ставит курсор после вставки — удобно, чтобы сразу набрать пробел и продолжить. |
| Default keymap | `<leader>mi` в markdown-буферах (`i` — insert). Регистрируется в `ftplugin/markdown.lua` рядом с `mf`/`ms` из M2. |
| Изоляция telescope от остальной логики | `notes.lua`, `insert.lua` ничего не знают про telescope: чистые функции на путях и строках, тестируемы юнит-тестами без UI. `picker.lua` — единственный модуль с `require("telescope...")`. |

## Шаги выполнения

Шаги упорядочены так, чтобы после каждого можно было запустить осмысленную проверку — тест или ручной сценарий.

### Шаг 1. Скаффолдинг

Создать заглушки для трёх новых модулей и зарегистрировать команду `:MdxInsertLink`.

#### Состав

- `lua/mdx/notes.lua` объявляет:

```lua
function M.collect(root, opts)  -- returns list of paths
function M.title(path)           -- returns string
```

- `lua/mdx/insert.lua` объявляет:

```lua
function M.format_link(target_path, source_dir, title)  -- returns string
function M.at_cursor(link_string)
```

- `lua/mdx/picker.lua` объявляет:

```lua
function M.open(opts, on_select)
```

На этом шаге функции возвращают пустые значения / `nil`. Цель — модули подгружаются, имена доступны.

#### Изменения в `lua/mdx/init.lua`

В `setup(opts)` дописать регистрацию команды:

```lua
vim.api.nvim_create_user_command("MdxInsertLink", function()
    require("mdx").insert_link()
end, {})
```

И добавить публичную функцию `M.insert_link()` со стабом — на этом шаге выводит `vim.notify("mdx: insert_link not implemented yet", ...)`.

**Готовность:** `:MdxInsertLink` выполняется без ошибок и выводит уведомление-заглушку; `:lua print(vim.inspect(require('mdx.notes')))` показывает таблицу с двумя функциями; то же для `mdx.insert` и `mdx.picker`.

### Шаг 2. Сборщик заметок

Реализовать рекурсивный обход дерева и фильтрацию.

#### Состав `lua/mdx/notes.lua`

```lua
function M.collect(root, opts)
```

Поведение:

1. Если `root` не задан — возвращает пустой список.
2. Через `vim.fs.find` ищет все файлы с суффиксом `.md` в `root`, рекурсивно (`limit = math.huge`, `type = "file"`).
3. Применяет фильтр исключений: пути, содержащие любую из подстрок `opts.exclude` (плюс встроенные `/.git/`, `/node_modules/`), отбрасываются.
4. Возвращает отсортированный список абсолютных путей.

`opts.exclude` — таблица строк-подстрок. Сравнение substring-based, не glob — простоты ради.

#### Тесты

`tests/notes_spec.lua` использует `vim.fn.tempname()` для создания изолированной директории, кладёт в неё `.md` и не-`.md` файлы, в т.ч. в подкаталогах, в т.ч. внутри `.git/`. Покрывает:

- одиночный файл в корне,
- файлы во вложенных подкаталогах,
- не-`.md` отбрасывается,
- `.git/` отбрасывается по умолчанию,
- пользовательские исключения через `opts.exclude`.

**Готовность:** все табличные кейсы зелёные.

### Шаг 3. Извлечение заголовка

Реализовать чтение первого `# H1` из файла с fallback на имя файла.

#### Состав `lua/mdx/notes.lua`

```lua
function M.title(path)
```

Поведение:

1. Открывает файл через `io.lines(path)`.
2. Читает построчно до первого совпадения с регулярным выражением `^#%s+(.+)$` (Lua-pattern). Возвращает захваченную группу.
3. Если файл прочитан целиком и `H1` не найден — возвращает `vim.fs.basename(path)` без расширения `.md` (через `:gsub("%.md$", "")`).
4. Если файл не открывается — возвращает `vim.fs.basename(path)` без расширения.

#### Тесты

`tests/notes_spec.lua` (расширение того же файла), сценарии:

- Файл с `# Заголовок` в первой строке,
- Файл с `# Заголовок` после frontmatter и пустых строк,
- Файл без `H1` — возвращает имя файла,
- Несуществующий путь — возвращает имя файла,
- Заголовок с пробелами по краям (`# Заголовок  `) — обрезается через `:match("^%s*(.-)%s*$")`.

**Готовность:** все кейсы зелёные.

### Шаг 4. Форматирование ссылки

Связать резолвер пути и заголовок в готовую строку `[title](relpath)`.

#### Состав `lua/mdx/insert.lua`

```lua
function M.format_link(target_path, source_dir, title)
```

Поведение:

1. Вычисляет `rel = vim.fs.relpath(source_dir, target_path)`. Если `rel == nil` (cross-volume или другая аномалия) — использует `target_path` как есть.
2. Если `rel` начинается не с `.` и не с `/` — добавляет префикс `./`. Это явный маркер «не подразумевать пакетный путь», стилистически близкий к привычному markdown.
3. Возвращает строку `"[" .. title .. "](" .. rel .. ")"`.

#### Тесты

`tests/insert_spec.lua` — табличные тесты на пары `(target, source_dir, title, expected)`:

- Тот же каталог: `("/p/notes/foo.md", "/p/notes", "Foo")` → `"[Foo](./foo.md)"`.
- Подкаталог: `("/p/notes/sub/foo.md", "/p/notes", "Foo")` → `"[Foo](./sub/foo.md)"`.
- Родительский каталог: `("/p/foo.md", "/p/notes", "Foo")` → `"[Foo](../foo.md)"`.
- Заголовок с символами markdown (`[Foo](bar)`) — оставляется как есть на M3 (escape отложен до отдельной фичи).

**Готовность:** все кейсы зелёные.

### Шаг 5. Вставка ссылки в буфер и сборка пайплайна

Реализовать `at_cursor` и связать `notes` + `insert` в публичный `M.insert_link()`.

#### Состав `lua/mdx/insert.lua`

```lua
function M.at_cursor(link_string)
```

Реализация: `vim.api.nvim_put({ link_string }, "c", true, true)`.

#### Изменения в `lua/mdx/init.lua`

```lua
function M.insert_link()
```

Поведение:

1. Получает `source_dir = vim.fs.dirname(vim.api.nvim_buf_get_name(0))`.
2. Вызывает `picker.open(config, on_select)`, где `on_select(entry)`:
    - Извлекает `title = entry.title`, `path = entry.path`.
    - Формирует `link = insert.format_link(path, source_dir, title)`.
    - Вставляет: `insert.at_cursor(link)`.

`config` собирается из `M.config` (Шаг 6 расширит его опциями `root`, `link_text`, `exclude`).

#### Готовность

На этом шаге `picker.open` ещё стаб — её реализация в Шаге 6. Поэтому `:MdxInsertLink` пока выводит уведомление о том, что picker не открыт. Цель Шага 5 — иметь рабочую цепочку «получить путь и заголовок → отформатировать → вставить» отдельно. Можно покрыть интеграционным тестом: руками подсунуть `entry`-словарь и проверить, что в текущий буфер вставляется ожидаемая строка.

### Шаг 6. Telescope-picker

Реализовать picker и связать со всем выше.

#### Состав `lua/mdx/picker.lua`

```lua
function M.open(opts, on_select)
```

Поведение:

1. Вычисляет `root` из `opts.root` (значение или результат вызова, если функция). Если `root == nil` — `vim.fs.root(0, { ".git", ".mdx", "stylua.toml" })` или `vim.fn.getcwd()` как fallback.
2. Собирает `paths = require("mdx.notes").collect(root, { exclude = opts.exclude })`.
3. Для каждого `path` вычисляет `title = require("mdx.notes").title(path)` (с учётом `opts.link_text`: `"filename"` форсит имя файла, `"title"` — `H1` без fallback на имя, `"auto"` — текущее поведение `notes.title`).
4. Формирует список `entries = { { path = ..., title = ... }, ... }`.
5. Открывает telescope через `pickers.new` с `finders.new_table` и `sorters.get_generic_fuzzy_sorter`.
    - Display: `string.format("%s (%s)", title, vim.fs.relpath(root, path))`.
    - На `<CR>`: вызвать `on_select(entry)`, закрыть picker.

Реализация на основе стандартного API telescope:

```lua
local pickers = require("telescope.pickers")
local finders = require("telescope.finders")
local conf = require("telescope.config").values
local actions = require("telescope.actions")
local action_state = require("telescope.actions.state")
```

#### Расширение `setup(opts)` в `lua/mdx/init.lua`

Добавить в `defaults`:

```lua
root = nil,             -- nil → автодетект; строка или функция
link_text = "auto",     -- "auto" | "title" | "filename"
exclude = {},           -- список substring-паттернов
keymaps = {
    follow = "<leader>mf",
    follow_split = "<leader>ms",
    insert_link = "<leader>mi",  -- новое
},
```

#### Изменения в `ftplugin/markdown.lua`

Добавить keymap для `insert_link` по тому же шаблону, что для `follow`/`follow_split`:

```lua
if config.keymaps.insert_link then
    vim.keymap.set("n", config.keymaps.insert_link, function()
        require("mdx").insert_link()
    end, { buffer = true, desc = "mdx: insert link to existing note" })
end
```

**Готовность:** `<leader>mi` или `:MdxInsertLink` в открытом `.md`-файле открывает telescope-picker со списком заметок проекта; выбор заметки вставляет в позицию курсора корректную markdown-ссылку с относительным путём и заголовком.

## Критерии готовности M3

1. `nvim --headless` с `PlenaryBustedDirectory nvim/tests/` зелёный (включая новые `notes_spec.lua` и `insert_spec.lua`).
2. В открытом `.md`-файле `<leader>mi` открывает telescope-picker; список содержит все `.md`-заметки проекта, исключая `/.git/` и встроенные паттерны.
3. Выбор заметки в picker'е вставляет в позицию курсора строку вида `[Заголовок](./относительный/путь.md)`. Курсор после вставки стоит сразу за `)`.
4. На заметке без `H1` `text`-часть равна имени файла без расширения.
5. `setup({ link_text = "filename" })` принудительно использует имя файла даже при наличии `H1`.
6. `setup({ root = "/иной/путь" })` ограничивает picker этим каталогом.
7. `setup({ exclude = { "/_drafts/" } })` исключает файлы из соответствующего поддерева.
8. `setup({ keymaps = { insert_link = false } })` отключает keymap; команда `:MdxInsertLink` остаётся работать.
9. README содержит примеры использования и переопределения настроек.

После выполнения этих пунктов M3 закрыт; пара «создать ссылку — перейти по ссылке» работает целиком в neovim.
